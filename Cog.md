# Cog — 认知模型

## 实体

| 实体 | 类型 | 说明 |
|------|------|------|
| 老师 | HI | 唯一操作者，选班级、选课题、抽学生、判通过/不通过 |
| Claude | AI | 协助开发和维护代码 |
| 学生 | Info | 被抽选对象，按班级分组，有姓名 |
| 班级 | Info | 高一8班、12班、16班、20班 |
| 课题 | Info | 7个哲学课题，每个课题下有若干问题 |
| 问题 | Info | 归属于课题，抽背时展示给学生回答 |
| 抽背记录 | Info | 每次抽背生成一条：日期、时间、班级、课题、学生、问题、结果 |
| 关注名单 | Info | 按"班级+课题"存储，标记需要优先抽选的学生 |

## 关系

```
老师 ──选择──→ 班级 + 课题
老师 ──抽选──→ 学生 + 问题
老师 ──判定──→ 通过 / 不通过 ──→ 生成抽背记录
老师 ──设置──→ 关注名单 ──→ 影响抽选优先级
```

## 双文件架构

```
超级抽背大师.html（教室版）
  ├── 写入：抽背记录
  └── 读取：关注名单（决定抽选优先级）

管理后台.html（教师版）
  ├── 读取：抽背记录（学情看板展示）
  └── 读写：关注名单（设置哪些学生被关注）
```

两个文件通过 localStorage 共享数据，互不依赖。

## 数据存储

- 抽背记录：localStorage，键名 `chouBeiDaShi_records`
- 关注名单：localStorage，键名 `chouBeiDaShi_focus`
