# Spec — 规约文档

## 文件架构（数据代码分离）

```
项目目录/
├── 超级抽背大师.html      ← 教室用，只有抽背功能，干净简洁
├── 管理后台.html          ← 老师用，学情看板 + 关注名单 + 数据管理
├── data.js                ← 共享数据（学生名单 + 课题问题 + 分组），改题库只改这个文件
└── questions/             ← MD格式题库（数据源，便于编辑复用）
    ├── 第一课-复习.md
    ├── 1.1-追求智慧的学问.md
    ├── ...
    └── 6.3-价值的创造和实现.md

U盘/
├── 超级抽背大师.html
└── 管理后台.html
```

`data.js` 是数据源文件，更新题库时 Claude 读取 `data.js` 并将数据同步写入两个 HTML（内联 `<script>` 中）。两个 HTML 是自包含的，用 `file://` 协议直接打开即可工作。两个文件共享同一 localStorage，数据互通。

## 用户故事（MAS）

### MAS-1：不再靠举手（光明故事）
- 起点：老师问"谁来背"没人举手，课堂冷场
- 转折：用工具随机抽选，滚动动画制造紧张感
- 终点：每个学生都可能被抽到，课堂参与度提升

### MAS-2：重点学生不漏掉（光明故事）
- 起点：总有几个学生侥幸没被抽到，薄弱环节暴露不出来
- 转折：设置关注名单，系统优先抽选
- 终点：薄弱学生得到更多练习机会

### MAS-3：学情有据可查（光明故事）
- 起点：学期末问"谁背得差"，只能凭印象
- 转折：每次通过/加油自动记录，看板可视化
- 终点：打开管理后台就知道谁需要关注

### MAS-4：数据不丢失（光明故事，v2.3.0 已解决）
- 起点：正常使用，数据积累了一学期
- 转折：教室电脑重装系统 / 换电脑
- 解决：管理后台"数据管理"页提供导出备份/导入备份，一键保存和恢复全部数据

### MAS-5：多教室数据汇总（光明故事，v3.0.1 已解决）
- 起点：在不同教室（8班、12班、16班、20班）抽背，每台电脑数据独立
- 问题：回办公室看学情看板，只有本机数据，看不到教室的记录
- 解决：教室版配置页底部"导出抽背记录"一键导出 JSON → 带回办公室 → 管理后台导入合并（自动去重）

## 各页面功能

### 教室版 — 配置页
- 选择班级（下拉框）
- 选择课题（两级分组下拉框，用 `<optgroup>` 按课分组）
- "上课"按钮 → 进入候课页

### 教室版 — 候课页
- 显示当前课题的全部问题列表（学生看屏幕背诵3分钟）
- 显示问题列表（无答案），学生看屏幕自主背诵
- "开始抽背"按钮 → 进入抽背页

### 教室版 — 抽背页
- 顶栏：返回按钮 + ◀▶箭头切题 + 题号小圆点指示器（蓝=未问，绿=已问，当前题外圈高亮）
- 问题文字区：全宽白色卡片，字号3vw，后排9米可读
- 学生区：深色背景，大名字8vw，滚动动画2秒后停下
- 抽选逻辑：关注名单中的学生优先，抽完再抽其他人
- 通过/加油后回到抽选状态，题目不变（同一题可多抽几人）
- 点箭头才切题，老师完全掌控节奏

### 管理后台 — 首页
- 功能导航：学情看板、关注名单、数据管理

### 管理后台 — 学情看板
- 顶部：班级筛选 + 导出CSV + 清空数据
- 三个标签页切换：
  - 按班级：总览卡片 + 各班通过率条形图
  - 按学生：表格（姓名、班级、被抽次数、通过次数、通过率），按通过率升序
  - 按课题：各课题通过率条形图
- 纯CSS条形图，不依赖外部库

### 管理后台 — 关注名单
- 选择班级 + 课题（下拉框，动态生成）
- 点击学生姓名切换关注状态（橙色 = 已关注）
- 显示已关注人数
- 可清空当前班级+课题的关注名单

### 管理后台 — 数据管理
- 导出备份 / 导入备份（JSON格式，含全部数据）
- 两个标签页：班级名单、课题问题
- 左侧列表选择 + 右侧 textarea 编辑
- 支持新增/删除班级和课题
- 从 Excel 复制粘贴名单，保存时自动去空行和首尾空格
- 数据存入 localStorage，教室版自动读取

## 版本管理规则（语义化版本 SemVer）

格式：`vX.Y.Z`（主版本.次版本.修订号）

| 位置 | 名称 | 什么时候 +1 | 举例 |
|------|------|------------|------|
| X | 主版本 | 大改：加新页面、改核心架构 | v1→v2：加了看板和记录 |
| Y | 次版本 | 新功能：加按钮、加新模式 | v2.0→v2.1：加关注名单 |
| Z | 修订号 | 修bug、调样式、删元素 | v2.0.0→v2.0.1：修引号bug |

递增规则：
- Z 递增时，Y 不变（v2.0.1 → v2.0.2）
- Y 递增时，Z 归零（v2.0.3 → v2.1.0）
- X 递增时，Y和Z 都归零（v2.1.0 → v3.0.0）

## 使用工作流（老师日常）

### 上课抽背
1. 插 U 盘 → 双击打开 `超级抽背大师.html`
2. 选班级 + 课题 → 上课 → 抽背
3. **下课前**：回到配置页 → 点底部"导出抽背记录" → 保存 JSON 文件到 U 盘
4. 拔 U 盘走人

### 查看学情（回办公室）
1. 打开管理后台 → 数据管理 → 导入备份
2. 选教室导出的 JSON 文件 → 自动合并（不覆盖、自动去重）
3. 多个班级的文件逐个导入即可
4. 切到学情看板查看汇总数据

### 数据说明
- 抽背记录存在每台电脑的浏览器 localStorage 里，各电脑独立
- 教室版"导出抽背记录"导出的是该电脑上的全部记录（JSON格式）
- 管理后台"导入备份"会智能合并：按 日期+时间+班级+学生+问题 去重，不会产生重复
- 同一个文件重复导入不会出问题

### 编辑题库（MD数据源）
1. 在 `questions/` 目录下编辑 MD 文件
2. 复习课格式：`1. 问题内容`（每行一题，无答案）
3. 新课格式：`Q：问题？` + `A：答案。`（Q/A 成对）
4. 让 Claude 执行编译：读取 MD → 生成新的 `data.js`
5. 复制两个 HTML 到 U 盘
