# 超级抽背大师 · 项目规约

## Real（现实约束）

- 纯 HTML 单文件，不依赖网络、不依赖任何外部库
- 必须在 file:// 协议下正常运行（U盘直接双击打开）
- 数据用 localStorage 保存，Chrome/Edge/Firefox 均支持
- 界面必须简洁优雅，不堆砌元素，能不加UI就不加

## Cog（认知模型）

系统中的实体和关系：

- **老师**（HI）：操作者，选班级、选课题、抽学生、判通过/不通过
- **学生**（Info）：被抽选对象，按班级分组，有姓名
- **班级**（Info）：高一8班、12班、16班、20班
- **课题**（Info）：7个哲学课题，每个课题下有若干问题
- **抽背记录**（Info）：日期、时间、班级、课题、学生、问题、结果
- **关注名单**（Info）：按"班级+课题"存储，标记优先抽选的学生

关系：
- 老师 → 选择 → 班级 + 课题
- 老师 → 抽选 → 学生 + 问题
- 老师 → 判定 → 通过/不通过 → 生成记录

## Spec（功能规约）

### 页面流程

```
配置页 → 候课页 → 抽背页
  ↘ 学情看板页
```

### 当前功能

- 配置页：选班级、选课题、进入上课或学情看板
- 候课页：展示本课题全部问题 + 关注名单设置
- 抽背页：左侧选问题、右侧抽学生、通过/不通过按钮
- 学情看板：按班级/按学生/按课题三维度统计 + CSV导出
- 智能抽选：关注名单学生优先，抽完再抽其他人

### JS中的中文引号规则

题目数据中如果包含中文双引号，必须用 Unicode 转义：
- 左引号 `"` → `\u201c`
- 右引号 `"` → `\u201d`

不能直接写中文引号，否则会与 JS 字符串定界符冲突导致整个脚本崩溃。

## 版本管理规则（语义化版本 SemVer）

格式：`vX.Y.Z`（主版本.次版本.修订号）

| 位置 | 名称 | 什么时候 +1 | 举例 |
|------|------|------------|------|
| X | 主版本 | 大改：加新页面、改核心架构 | v1→v2：加了看板和记录 |
| Y | 次版本 | 新功能：加按钮、加模式 | v2.0→v2.1：加关注名单 |
| Z | 修订号 | 修bug、调样式、删元素 | v2.0.0→v2.0.1：修引号bug |

规则：
- Z 递增时，Y 不变（v2.0.1 → v2.0.2 → v2.0.3）
- Y 递增时，Z 归零（v2.0.3 → v2.1.0）
- X 递增时，Y和Z 都归零（v2.1.0 → v3.0.0）
- 每次改代码后必须更新 title 标签中的版本号
- Git commit message 以版本号开头
